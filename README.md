# Cassette2Vec-EC

**Cassette-aware genomic machine learning for *Enterococcus cecorum* pathogenicity prediction**

[![DOI](https://zenodo.org/badge/DOI/10.5281/zenodo.18529389.svg)](https://doi.org/10.5281/zenodo.18529389)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)
[![Python 3.11](https://img.shields.io/badge/python-3.11-blue.svg)](https://www.python.org/downloads/)

## Overview

Cassette2Vec-EC is a structural genomics framework that converts annotated *Enterococcus cecorum* genomes into cassette units (local gene neighborhoods anchored in genomic islands), encodes each cassette as a fixed-length feature vector, and applies genome-grouped machine learning to predict pathogenic lineages while preventing within-genome leakage.

**Key performance (GroupKFold-by-genome, n=145 genomes):**
- AUROC: 0.975 ± 0.030
- Average Precision: 0.938 ± 0.077
- Brier Score: 0.056 ± 0.058

**Associated manuscript:**  
Lagad RR, Rafi S, Goswami A. *Cassette2Vec-EC: cassette-aware genomic ML for Enterococcus cecorum pathogenicity prediction.* Microbial Genomics (submitted 2026).

## Reproducing the Analysis (Step-by-Step)

### Prerequisites

External tools required for feature engineering from raw genomes:
- [Prokka v1.14.6](https://github.com/tseemann/prokka) — genome annotation
- [IslandViewer 4](https://www.pathogenomics.sfu.ca/islandviewer/) — genomic island prediction
- [ABRicate v1.0.1](https://github.com/tseemann/abricate) — AMR screening (ResFinder + CARD databases)
- [PIRATE v1.0.4](https://github.com/SionBayliss/PIRATE) — pangenome ortholog clustering

> **If you only want to reproduce the figures and model evaluation**, skip to Step 3 — all required files are on Zenodo.

---

### Step 1 — Feature Engineering (raw genomes → feature matrix)

Run notebook `01_feature_engineering.ipynb` or follow the pipeline:

```bash
# 1a. Annotate genomes with Prokka
for fna in data/genomes/*.fna; do
    prokka --outdir prokka_out/$(basename $fna .fna) \
           --prefix $(basename $fna .fna) \
           --kingdom Bacteria --genus Enterococcus $fna
done

# 1b. Run IslandViewer (web interface: https://www.pathogenomics.sfu.ca/islandviewer/)
#     Download CSV output per genome → place in data/islandviewer/

# 1c. Run ABRicate against ResFinder and CARD
abricate --db resfinder data/genomes/*.fna > data/abricate/abricate_all_merged.csv
abricate --db card      data/genomes/*.fna >> data/abricate/abricate_all_merged.csv

# 1d. Run PIRATE for pangenome clustering
PIRATE -i data/gff3/ -o pirate_out/ -t 8

# 1e. Build the cassette feature matrix
python cassette2vec_predict.py --batch \
    --islandviewer_dir data/islandviewer/ \
    --abricate_dir     data/abricate/ \
    --model            data/cassette2vec_v11_model.pkl \
    --outdir           predictions/
```

---

### Step 2 — Add AMR Neighborhood Features

This step computes `AMR_neighborhood_score` and `AMR_cluster_size` for each gene, required for the SHAP analysis.

```bash
python add_AMR_neighborhood_v1.py \
    --input  data/cassette2vec_ML_features_v1_with_mobility_load.csv \
    --output data/cassette2vec_ML_features_v1_with_AMR_neighborhood.csv
```

Expected output: `data/cassette2vec_ML_features_v1_with_AMR_neighborhood.csv`  
Runtime: ~5–15 minutes depending on data size.

python cassette2vec_ML_v11_FINAL.py \

### Step 4 — Predict a New Genome

```bash
# Single genome
python cassette2vec_predict.py \
    --genome_id    GCF_NEW_GENOME.1 \
    --islandviewer data/islandviewer/GCF_NEW_GENOME_islands.csv \
    --abricate     data/abricate/GCF_NEW_GENOME_abricate.csv \
    --model        data/cassette2vec_v11_model.pkl \
    --outdir       predictions/

# Batch mode (multiple genomes at once)
python cassette2vec_predict.py --batch \
    --islandviewer_dir data/islandviewer/ \
    --abricate_dir     data/abricate/ \
    --model            data/cassette2vec_v11_model.pkl \
    --outdir           predictions/
```

**Output:** A risk score (0–1) per genome, cassette-level scores, and a ranked list of the top high-risk cassette loci for follow-up.

## Model Features

The trained model uses 17 active numeric features (3 schema fields are reserved for future versions):

| Feature | Description |
|---------|-------------|
| `GI_flag` | Gene is located within a predicted genomic island |
| `amr_hit` | ABRicate AMR hit indicator |
| `mobility_score` | Mobility marker flag (integrase/transposase/recombinase) |
| `start`, `end` | Genomic coordinates of the gene |
| `island_start`, `island_end` | Coordinates of the merged GI interval |
| `island_length` | Length of the GI interval in bp |
| `integrase_flag` | Integrase annotation indicator |
| `transposase_flag` | Transposase/IS element annotation indicator |
| `recombinase_flag` | Recombinase annotation indicator |
| `Mobility_Load` | Sum of mobility-associated genes in the cassette |
| `GI_AMR_density` | Fraction of GI genes that are AMR-positive |
| `AMR_cluster_size` | Number of AMR genes in the ±3-gene window |
| `AMR_neighborhood_score` | Composite score of mobility context around AMR genes |
| `island_gene_count` | Total gene count in the GI interval |
| `amr_gene_count_in_island` | Count of AMR genes in the GI interval |

## Reproducibility

All results in the manuscript are fully reproducible:

- **Fixed random seed:** `SEED = 42` in all scripts and notebooks
- **Genome-grouped evaluation:** `GroupKFold(n_splits=5)` — no within-genome leakage
- **Pinned dependencies:** `requirements.txt` and `environment.yml`
- **Pre-built model and data:** Available on Zenodo ([DOI: 10.5281/zenodo.18529389](https://doi.org/10.5281/zenodo.18529389))
- **GitHub release:** Tag `v1.1.0`, commit hash updated at publication
